# TA-santa

A Splunk Technology Add-on (TA) to collect, parse, and analyze data from [Santa](https://github.com/northpolesec/santa), a binary whitelisting/blacklisting system for macOS developed by NorthPoleSec.

## Overview

This TA ingests and normalizes logs generated by Santa, enabling Splunk users to monitor macOS application whitelisting activity, detect policy violations, and gain insights into endpoint security.

The add-on is based on the open source Santa project maintained here: https://github.com/northpolesec/santa

## Features

- Collects Santa log data from macOS endpoints
- Parses Santa logs into structured Splunk events
- Extracts key fields for searching and reporting
- Enables detection of blocked or flagged binaries
- Supports integration with Splunk Enterprise Security for endpoint monitoring

## Installation

1. Clone or download this TA-santa repository.
2. Install the TA on your Splunk instance via **Manage Apps > Install app from file**.
3. Configure inputs to collect Santa logs from your macOS devices.
4. Restart Splunk to enable the add-on.

## Configuration

- Configure your endpoints to forward Santa logs to Splunk (via syslog, Universal Forwarder, or other method).
- Set up inputs in the TA to parse and index incoming Santa log data.

## Usage

- Use the provided field extractions and source types to search Santa events.
- Build alerts or dashboards to monitor application execution policies and anomalies.
